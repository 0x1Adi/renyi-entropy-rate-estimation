Multi-Scale Collision Counting for Rényi Entropy Rate Estimation

Aditya Tiwari · Independent Researcher, Goa, India

Code Paper Dataset Try It

TL;DR

We measure how "patterned" a byte stream is using Rényi collision entropy instead of Shannon entropy. By looking at collision rates at multiple scales (1-gram, 2-gram, 3-gram…) and fitting a line, we get an entropy rate that is dramatically better at telling DNS tunnel tools apart — ARI = 0.720 vs Shannon's 0.204 — with zero machine learning. Just count, fit, classify.

Background

Two Ways to Measure Randomness

Given a stream of data, we want a single number that captures "how random is this?" There are two classic approaches, and they weight the same probabilities differently.

H₁

Shannon Entropy

H₁ = −Σ pᵢ log₂ pᵢ

Weights each symbol by −p log p. This function peaks near p ≈ 0.37, so medium-probability events dominate. Great for compression, but dilutes the signal from common patterns.

Weight: −p log p
F₂

Rényi Collision Entropy

H₂ = −log₂(Σ pᵢ²)

Weights each symbol by p². High-probability symbols dominate quadratically. This amplifies exactly the peaked patterns that encoding tools (base32, base64, hex) create.

Weight: p²

Why does this matter for security?

DNS tunneling tools (iodine, dnscat2, dns2tcp…) encode arbitrary data into DNS query names using base32/base64/hex. Each encoding creates a distinctive nibble transition pattern — a few k-grams appear far more often than others.

Shannon entropy spreads its attention uniformly. Rényi's p² weighting amplifies these peaks, making each tool's encoding signature stand out. The multi-scale slope then captures how these peaks evolve across 1-grams → 2-grams → 3-grams, giving us a per-tool fingerprint.

Why nibbles?

At byte level (α=256), you need >65K samples just for 2-gram statistics to converge. At nibble level (α=16), 2-grams need only 256 bins and 3-grams need 4,096 — feasible from just 5KB of data.

Why multi-scale?

Single-scale Rényi at byte level scores ARI = 0.124 (worse than Shannon!). The slope across scales is what captures sequential structure. Without multi-scale: no advantage.

The math behind the slope

For a Markov chain with transition matrix P, define M₂ by squaring each entry: (M₂)ᵢⱼ = Pᵢⱼ². The k-gram collision probability is:

F₂(k) = v⊤ · M₂ᵏ⁻¹ · 𝟏    where vᵢ = πᵢ²

By the Perron–Frobenius theorem, log F₂(k) becomes linear in k for large k, with slope = log ρ(M₂). So a simple OLS regression gives us the entropy rate: h₂ = −slope / ln 2.

Key takeaways

  • Single-scale Rényi is worse than Shannon (row 2 vs 1). The multi-scale slope is what unlocks the advantage.
  • Debiasing doesn't explain the gap (rows 3a = 3b). Miller-Madow correction changes nothing for Shannon.
  • Adding h₃, h₄ helps modestly (+0.027 ARI). The main signal is in h₂ alone.
  • No model training required. Just reference centroids from labeled examples — nearest-centroid classification.
